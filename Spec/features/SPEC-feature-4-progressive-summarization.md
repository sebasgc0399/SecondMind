## Objetivo

Cuando Sebastián abre una nota vieja, ve de un vistazo los pasajes clave (**bold**, L1), los esenciales (<mark>highlight amarillo</mark>, L2) y un resumen ejecutivo arriba (L3). El badge del header le dice el nivel de destilación, con un tip contextual que le indica la próxima acción. La técnica es acumulativa (cada capa se suma) y no destructiva: si borra el resumen, el nivel baja al correspondiente a las marcas restantes.

## Decisiones clave

1. **Hook unificado, no dos hooks paralelos** (corrección del review). Un solo `useNoteSave` maneja editor + summary con un timer y un `save()` atómico. Previno race de writes concurrentes con `distillLevel` stale.
2. **Optimistic `setPartialRow` ANTES de `await updateDoc`** (cambio de orden respecto al flow previo). Si `updateDoc` falla, `pendingRef` vuelve a true y el retry re-escribe los mismos datos (idempotente). Consistente con `useHabits.toggleHabit`.
3. **Popover de `@base-ui/react/popover` en vez de dropdown manual.** Mismo paquete que Dialog (ya ampliamente usado), ahorra ~60 líneas de click-outside/escape/portal manuales.
4. **Slash menu item "Resaltar" descartado** (desviación del SPEC). UX rota porque los slash commands disparan sin selección. Única vía a L2: `Ctrl+Shift+H`.
5. **`setState` dentro de `setTimeout(0)`** en el `useEffect` de auto-open del colapsable. Regla `react-hooks/set-state-in-effect`; patrón ya usado en `useHybridSearch`.
6. **No se tocó el schema de TinyBase ni `types/note.ts`** — estaban preparados desde Fase 1. La feature agregó solo lógica de compute/persist/render.
7. **`summaryL1`/`summaryL2` dead weight intencional**. El schema los tiene como strings; ninguna feature actual los usa (L1/L2 se derivan de marks en `content`). Registrado como tech-debt potencial; no tocar ahora.

## Lecciones

- **Slash commands disparan sin selección activa.** Cualquier comando que necesite un rango (toggleHighlight, toggleLink, marcas en general) en un slash menu es no-op silencioso → UX rota ("nada pasa al seleccionar el item"). Los slash items válidos son block-level (heading, list, divider). Para marks usar bubble menu sobre selección o shortcut directo.
- **Race "colapsable cerrado + focus": `setIsOpen(true) → requestAnimationFrame → focus()`.** Sin el rAF, `ref.current` es null porque el elemento no está en DOM todavía cuando React aún no aplicó el setState. Patrón aplicable a cualquier focus disparado desde un trigger externo hacia un input dentro de un container condicional (modal, dropdown, accordion, drawer).
- **`@base-ui/react/popover` sigue exactamente el mismo patrón que el Dialog ya usado en 7 archivos:** `Root → Trigger → Portal → Positioner → Popup` + data-attrs `data-starting-style` / `data-ending-style` para animaciones. Replicable directamente a tooltip/menu/select del mismo paquete sin re-aprender API.
- **Auto-resize de textarea: `el.style.height = '0px'` antes de leer `scrollHeight`, NO `'auto'`.** En iOS Safari, `'auto'` deja el browser eligiendo una altura intermedia que rompe la medición. `0px` fuerza el shrink completo y el `scrollHeight` posterior es confiable.
- **Schema preparado pero campos no usados son tech-debt registrable, no removible.** Borrar `summaryL1`/`summaryL2` del schema TinyBase rompe el rollback de cualquier nota persistida en Firestore que ya tenga esos keys serializados. YAGNI aplica hacia adelante (no agregar features especulativas), no hacia atrás (no quitar fields ya en producción aunque estén vacíos).
