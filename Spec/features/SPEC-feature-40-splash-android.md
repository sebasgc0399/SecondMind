## Objetivo

Reemplazar el splash legacy (color violeta `#878bf9` sin logo, solo el ícono del launcher Android genérico) por un branded splash adaptativo. El usuario ve el branding de SecondMind desde el primer frame de carga.

## Qué se implementó

- **F40 — System splash adaptativo (Capa 1, Android 12+ SplashScreen API):** se migró `AppTheme.NoActionBarLaunch` de `android:background="@drawable/splash"` legacy a parent `Theme.SplashScreen` con las 3 attrs canónicas (`windowSplashScreenBackground=@color/splashBackground`, `windowSplashScreenAnimatedIcon=@drawable/ic_launcher_foreground` vector extraído de `public/favicon.svg`, `postSplashScreenTheme=@style/AppTheme.NoActionBar`). Fondo `#ffffff` light / `#0a0a0a` dark via `colors.xml` + `values-night/colors.xml`. Se eliminó el `<layer-list>` violeta legacy. Archivos tocados: `android/app/src/main/res/values/styles.xml`, `android/app/src/main/res/values/colors.xml`, `android/app/src/main/res/values-night/colors.xml`, `android/app/src/main/res/drawable/splash.xml` (eliminado), `assets/logo.png`, `capacitor.config.ts`.
- **F40 — Branded React splash `<AppBootSplash>` (Capa 2):** pantalla React post-bootstrap con fondo `bg-background` (light/dark), logo `favicon.svg` 174px centrado absoluto y texto `<h1>SecondMind</h1>` debajo. El handoff system splash → AppBootSplash quedó seamless (mismo tamaño/posición de icon entre frames). El tamaño 174px se calibró empíricamente. Archivos tocados: `src/components/layout/AppBootSplash.tsx`.
- **F40 — Lifecycle del splash:** `useHideSplashWhenReady` dispara `hideSplash()` en `useEffect(..., [])` del Layout (primer mount, no fin del bootstrap); `bootSplashMinElapsed` (timeout 800ms) garantiza perceptibilidad en cold starts rápidos; safety timeout 5s en `main.tsx` para crash pre-mount; `hideSplash()` idempotente vía flag module-scope. Archivos tocados: `src/lib/splash.ts`, `src/hooks/useHideSplashWhenReady.ts`, `src/app/layout.tsx`, `src/main.tsx`.
- **F40 — Recovery de iconos del launcher:** `@capacitor/assets generate` sobrescribió `mipmap-anydpi-v26/ic_launcher{,_round}.xml` apuntando a PNGs rasterizados (degradación visual API 26+); restaurados con `git checkout HEAD -- mipmap-anydpi-v26/` para preservar el adaptive icon vector drawable. Los 12 `splash.png` generados quedaron committeados pero inertes (la API SplashScreen usa el vector `windowSplashScreenAnimatedIcon`, no los PNG).

## Lecciones

- **El branded splash "completo" (logo + texto) no es alcanzable con la SplashScreen API de Android 12+ — solo icon centrado sobre color.** Para un splash con layout libre, el system splash cubre el primer frame con icon centrado y una capa React post-bootstrap entrega el branding; el handoff debe ser seamless (mismo tamaño/posición de icon entre los dos frames). Generaliza a cualquier branded splash nativo Android.
- **El `hide()` del system splash debe dispararse al primer mount del Layout, no al fin del bootstrap** — si no, el system splash tapa la capa React branded durante todo el bootstrap y el handoff se vuelve invisible. Apenas React puede pintar algo (incluso con `isLoading=true`), conviene ceder el frame.
- **El tamaño visible del icon en algunos OEMs (Samsung) no sigue la teoría del masking circular** — sin `windowSplashScreenIconBackgroundColor` no hay masking, el icon va al canvas 288dp completo. La calibración del tamaño en la capa React (para matchear el system splash) solo es posible empíricamente sobre device real.
- **`@capacitor/assets generate` sobrescribe `mipmap-anydpi-v26/ic_launcher{,_round}.xml`** degradando el adaptive icon vector a PNGs rasterizados (API 26+): exige recovery proactivo `git checkout HEAD -- mipmap-anydpi-v26/` tras correr la tool.
- **Samsung con DUAL_APP/Secure Folder: `adb install`/`am start` default van a user 150, no user 0** — pasar `--user 0` explícito. (QA validado en Galaxy R5CW30T2FDV.)
- Estos 5 gotchas están escalados a `Spec/gotchas/capacitor-mobile.md` (indexados en `Spec/ESTADO-ACTUAL.md` § "Capacitor Mobile") — consultar ahí el canon operativo vigente.
