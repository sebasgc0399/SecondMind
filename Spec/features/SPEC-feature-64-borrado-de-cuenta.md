## Objetivo

Un usuario puede **borrar su cuenta y TODOS sus datos personales de forma irreversible** desde la propia app: se reautentica, confirma, y un callable server-side arrasa su contenido + secretos + rastro de PII + el usuario de Auth. En **web** el flujo es completo in-app. En **Android (Capacitor)** el botón abre la danger zone web en el navegador del sistema (entrada in-app que Play exige). En **desktop (Tauri)** el botón se oculta en v1 (Tauri no se distribuye por Play ni está en el critical path → fast-follow). Cierra el gap GDPR/CCPA de borrado y satisface el requisito de Data deletion de Google Play.

## Qué se implementó

- **F1 — callable `deleteAccount` (el corazón):** callable v2 **self-service** (gate = auth + reauth reciente server-side por `auth_time`, NO admin-only) que orquesta el wipe total con Admin SDK: `recursiveDelete(users/{uid})` (doc fantasma) + `recursiveDelete(userSecrets/{uid})` + `allowlist/{email}` + `accessRequests/{email}` (guard si el token no trae email) + `rateLimits` del uid (range query sobre `documentId()`) + `auth().deleteUser` ÚLTIMO (retry ×3 ante transient). Idempotente, no transaccional. Gate puro extraído a `reauthGate.ts` (fail-closed). 15ª CF. Archivos: `src/functions/src/account/deleteAccount.ts`, `reauthGate.ts`, `index.ts`. Tests: unit gate + E2E emulador (wipe del doc fantasma + idempotencia + gating).
- **F2 — lib cliente + reauth:** `reauthenticate()` ramifica por provider (Google popup / email-pw credential) y fuerza `getIdToken(true)` antes de devolver (token con `auth_time` fresco); wrapper `deleteAccount()` del callable; mapeo de errores en `cfError.ts` (`reauth-required`). Archivos: `src/lib/account.ts`, `src/lib/cfError.ts`.
- **F3 — danger zone web (`DeleteAccountSection`) + montaje:** sección destructiva en Settings; un modal que exige tipear el email exacto (freno anti-accidente) + password si es email/pw → `reauthenticate()` → `deleteAccount()` → `signOut()` + redirect a `/login`. i18n es/en aditivo. Archivos: `src/components/settings/DeleteAccountSection.tsx`, `src/app/settings/page.tsx`, `src/locales/{es,en}/translation.json`.
- **F4 — test del gate de control-flow + smoke doc:** test de componente (jsdom) que blinda la ÚNICA defensa del caso "auth_time fresco por login + reauth de re-confirmación falla" (reauth falla → `deleteAccount` NUNCA se invoca); F1 server-side no lo frena. El wipe real no se automatiza por browser (YAGNI; F1 lo cubre). Happy path destructivo = smoke manual pre-release. Archivos: `DeleteAccountSection.test.tsx`.
- **F5 — fork Android (Custom Tab) + rediseño danger zone:** render unificado por runtime (`isNative=isCapacitor()`) con comportamiento bifurcado: web abre el modal; Capacitor llama `openWebDeletion()` (`@capacitor/browser` `Browser.open` → Chrome Custom Tab a `app.getsecondmind.co/settings#delete-account`, donde el reauth web funciona); el `<AlertDialog>` no se monta en nativo (imposible montar el modal en el WebView). Tauri oculto. De paso se rediseñó la danger zone (card tonal con ícono `TriangleAlert`, patrón `AppInfoSection`). Archivos: `DeleteAccountSection.tsx`, `src/lib/account.ts` (`openWebDeletion`), `translation.json` (`nativeHint`), `package.json` (`@capacitor/browser`), `android/*.gradle` (registro del plugin).
- **Hardening post-review (Sebastián, línea por línea):** retry ×3 de `auth().deleteUser` (mitiga Auth huérfano con email/PII residual), `uid` en logs de éxito/fallo (auditoría sin PII de contenido), comentario de seguridad explícito en el range query. Archivos: `deleteAccount.ts`.

## Lecciones

- **`auth_time` del token como gate de reauth server-side en un callable**, con el force `getIdToken(true)` en el cliente tras el reauth — sin él, el `auth_time` queda stale y el gate rechaza un reauth válido (silent failure). El gate va ANTES de cualquier I/O y es fail-closed (auth_time ausente/no-finito = expirado). → `gotchas/cloud-functions-guards.md`.
- **El control-flow del cliente es la ÚNICA defensa** del caso "auth_time fresco por LOGIN (<5 min) + reauth de re-confirmación que falla": el gate server-side NO lo frena (el auth_time es legítimo por el login). Defensa = `await reauthenticate()` ANTES de `await deleteAccount()` + catch, blindado por test de componente.
- **Borrar `rateLimits/` por range query sobre `documentId()`** (docs sin campo uid): el bound `[uid__ … uid__ + U+FFFF)` es seguro SOLO porque los UIDs de Firebase son longitud-fija sin underscore → revisar ANTES de habilitar custom UIDs. → `gotchas/cloud-functions-guards.md`.
- **Auth huérfano = PII residual real, no auto-recuperable:** si `auth().deleteUser` falla tras borrar `allowlist`, el usuario queda bloqueado del login y su email queda vivo en Auth; recuperación por SOPORTE, no auto-retry del usuario. El retry ×3 lo mitiga, no lo garantiza.
- **Shell nativo abre URL web del sistema (Chrome Custom Tab) para flujos que necesitan contexto de navegador (reauth):** `@capacitor/browser` `windowName` es web-only; Android abre un Custom Tab que comparte la sesión de Chrome (no el WebView de la app). → `gotchas/capacitor-mobile.md`.
- **Smoke de un plugin Capacitor en el APK real vía CDP, sin pasar el login:** los globals de Capacitor están en cualquier pantalla del WebView (incl. `/login`); `adb forward` al socket DevTools + cliente CDP puro (Node ≥22) + `dumpsys activity` para confirmar el efecto nativo. → `gotchas/capacitor-mobile.md`.
- **Deploy de una feature irreversible:** functions ANTES del cliente que lo llama (sin ventana de CF inexistente); deploy-antes-de-merge con smoke sobre prod real (rollback antes de contaminar main si falla); el tag rebuildea Tauri **y** Android (no hay forma de taggear solo Android) → un `tauri:build` local pre-valida el compile antes de publicar el auto-update desktop.
