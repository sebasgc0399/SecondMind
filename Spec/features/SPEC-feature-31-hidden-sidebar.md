## Objetivo

Permitir al usuario ocultar completamente el sidebar en desktop (≥1024px) para maximizar el área de trabajo, con persistencia cross-device en `users/{uid}/settings/preferences.sidebarHidden`. El chrome de navegación se reemplaza por un TopBar minimalista. Tablet/mobile sin cambios — la decisión solo afecta desktop.

Solución: pref boolean `sidebarHidden` en `UserPreferences` (mismo shape que `distillIntroSeen` de F22). Layout consume `usePreferences()` y deriva `showSidebar`/`showTopBar` con AND-gate sobre `prefsLoaded` para evitar flash. TopBar nuevo en `src/components/layout/TopBar.tsx` reusa el favicon oficial + `<PendingSyncIndicator compact />` (prop existente de F30) + botón Buscar ⌘K + `<QuickCaptureButton />`. Gate condicional en `page.tsx` evita duplicación del QuickCaptureButton entre TopBar y header del Dashboard. Toggle vía Cmd/Ctrl+B con 3 guards (foco/auth/breakpoint) usando `event.code === 'KeyB'` (independiente del layout físico). Selector "Visibilidad del menú" en `/settings` con wrapper `hidden lg:block`. CommandPalette enriquecido con 4 destinos faltantes (Dashboard, Hábitos, Objetivos, Settings) para fallback de navegación completa.

## Qué se implementó

Guard defensivo `data?.sidebarHidden === true` en `parsePrefs` (mismo patrón que `distillIntroSeen`). 3 tests unitarios nuevos: default false sin pref persistida, true cuando data trae literal `true`, false con truthy no-boolean (string/número). Test "doc legacy" actualizado para incluir el campo en `toEqual`. 13 tests pasan en `preferences.test.ts` (10 originales + 3 nuevos).

- **F31.3 — Componente `TopBar` minimalista (`src/components/layout/TopBar.tsx`):** componente standalone `h-12` con `border-b border-sidebar-border bg-sidebar` (continuidad visual con sidebar). - **F31.6 — `SidebarVisibilitySelector` en /settings (`src/components/settings/SidebarVisibilitySelector.tsx`, `src/app/settings/page.tsx`):** componente nuevo siguiendo el patrón canónico de `TrashAutoPurgeSelector` (grid sm:grid-cols-2, cards con `aria-pressed`, ring activo, descripción). 2 opciones: Visible (sidebarHidden=false) con icon `PanelLeft`, Oculto (sidebarHidden=true) con icon `PanelLeftClose`. Click usa `setPreferences(uid, { sidebarHidden })` con guard `if (!user || value === current) return`. Sección "Visibilidad del menú" entre Apariencia y Papelera, con wrapper `hidden lg:block` — solo visible en desktop.

- **F31.5 — `useSidebarVisibilityShortcut` con 3 guards (`src/hooks/useSidebarVisibilityShortcut.ts`, `src/app/layout.tsx`):** hook nuevo que monta `window.addEventListener('keydown', ...)` con cleanup. Match: `(event.metaKey || event.ctrlKey) && event.code === 'KeyB' && !event.shiftKey && !event.altKey`. **`event.code` (no `event.key`) es independiente del layout físico** — Dvorak/AZERTY siguen funcionando; con `event.key.toLowerCase() === 'b'` el atajo se rompería en layouts no-QWERTY. 3 guards: foco (`document.activeElement?.matches('input, textarea, select, [contenteditable=""], [contenteditable="true"]')` cubre TipTap `.ProseMirror`, search inputs, selects), auth (`if (!user) return` previene `setPreferences(user.uid, ...)` con uid undefined), breakpoint (`if (breakpoint !== 'desktop') return`). Tras pasar guards: `event.preventDefault()` + `setPreferences(user.uid, { sidebarHidden: !current })`. Mountado una vez en `src/app/layout.tsx` después de `useStoreInit`.

- **F31.2 — `QUICK_ACTIONS` del CommandPalette enriquecido (`src/components/layout/CommandPalette.tsx`):** array pasa de 4 a 8 entries cubriendo TODA la nav lateral. ## Lecciones

- **Para shortcuts globales con modificador, `event.code` (no `event.key`) es la elección por defecto.** `event.key` representa el carácter producido — afectado por el layout físico (Dvorak, AZERTY) y por el IME. `event.code` representa la tecla física en posición QWERTY equivalente — independiente del layout. Para atajos como Cmd/Ctrl+B, Cmd/Ctrl+S, etc. donde la tecla física es lo que importa, `event.code === 'KeyB'` es robusto. Los shortcuts existentes del repo (Cmd+K en CommandPaletteProvider, Alt+N en QuickCapture) usan `event.key` — funcionan en QWERTY estándar pero romperían en layouts no-QWERTY. Patrón aplicable a cualquier futuro shortcut global; documentado en ESTADO-ACTUAL.

- **Guard `activeElement.matches(...)` antes de `preventDefault()` para shortcuts que podrían pisar atajos del editor.** TipTap (`.ProseMirror` con `contenteditable="true"`) mapea Mod+B a bold por default; cualquier shortcut global con la misma combinación pisaría el bold cuando el editor tiene foco. Selector canónico: `'input, textarea, select, [contenteditable=""], [contenteditable="true"]'` — cubre TipTap, search inputs nativos, textareas (QuickCapture), selects (consistencia con `inbox/process/page.tsx`). Sin el guard, `event.preventDefault()` cancela el bold AND togglea el sidebar — comportamiento confuso aunque "funcione" técnicamente.

- **Side-effect global con dependencia en sesión: usar `if (!user) return` defensivo.** Hooks como `useSidebarVisibilityShortcut` se montan en `layout.tsx` dentro del Provider tree pero potencialmente antes que `useStoreInit` complete o que la sesión Firebase esté lista. Llamar `setPreferences(user.uid, ...)` con `user = null` rompe con uid undefined. Guard explícito al inicio del handler protege contra el race sin agregar API surface. Aplicable a cualquier handler async/global que dependa del uid en runtime.

- **Reusar prop existente del componente (F30 `<PendingSyncIndicator compact />`) en lugar de crear variante.** Detectado durante el cambio de scope D5/G5 a in-scope: el componente ya soportaba `compact` con chip 44×44 y tooltip via `title` — costo de incluirlo en el TopBar fue 1 línea JSX. La regla "1 línea + paridad funcional = no es deuda, es scope" aplica: cuando el componente ya tiene el modo necesario, sacrificarlo "porque típicamente son 0-1 entries" es la falacia que F28/F29 cerraron. El silent loss no se mide por frecuencia sino por consecuencia.

- **Asset oficial del proyecto antes que icono genérico de la lib.** Detectado por Sebastián durante E2E: el TopBar inicial usaba `<Brain />` de lucide cuando el repo ya tenía el favicon oficial en `public/favicon.svg` (mismo declarado en `index.html`). Vite sirve `public/` como `/`, así que `<img src="/favicon.svg" />` reusa el branding existente sin importar nada. Aplicable a cualquier asset visual de chrome global: revisar `public/` antes de buscar un icono genérico en lucide.

- **Riesgo de regresión silenciosa: gate condicional de elementos compartidos entre chromes alternos.** Cuando un componente CTA vive en dos lugares mutuamente exclusivos (Dashboard header con sidebar visible, TopBar global con sidebar oculto), el gate condicional para evitar duplicación es 1 línea — pero olvidarlo crea bug silencioso (sin TS error, sin runtime). Plan agent en step 2 SDD lo flagueó como riesgo #1, sumado al checklist E2E como check explícito. Patrón generalizable: cualquier feature que mueve un elemento entre chromes debe documentar en SPEC el check anti-duplicación.

## Out of scope (deuda explícita)

- **Animación de transición sidebar↔TopBar.** Swap es instantáneo. `tw-animate-css` ya en repo permitiría sumar `animate-in slide-in-from-left` etc. Follow-up trivial post-merge si se pide.
- **Toggle desde Command Palette.** Sumar "Ocultar sidebar" / "Mostrar sidebar" como quick action es 2 entries más. v1 con shortcut + Settings alcanza.
- **Hover edge / FAB para des-ocultar.** Solo ⌘B + Settings. Sin botón "Mostrar menú" en TopBar (logo lleva al Dashboard pero no toggea).
- **`getModKey()` util cross-platform** para renderizar `⌘K` en mac y `Ctrl K` en win/linux dinámicamente. Hoy hardcodeamos `⌘K` (paridad con `Alt+N` de QuickCaptureButton). Follow-up si llega feedback de UX.
- **Persistencia local-first del sidebar mientras `isLoaded=false`.** Default a sidebar visible pre-snapshot genera flash inverso (~100-300ms) para usuarios que persisten `sidebarHidden=true`. Mitigación futura: cachear última pref en `localStorage` como hint sincrónico. Aceptable para v1.
- **Atajo "force-show" para abrir solo en estado oculto.** Solo hay toggle bidireccional. Si surge demanda de "mostrar siempre" sin riesgo de ocultar accidentalmente, separar el shortcut.
- **Shortcuts existentes (Cmd+K, Alt+N) migrar a `event.code`.** Funcionan en QWERTY estándar; refactor proactivo no urgente. Si surge feedback de usuarios Dvorak/AZERTY, migrar.
