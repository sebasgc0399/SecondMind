## Objetivo

SecondMind tenía una paleta neutra gris (tokens oklch sin chroma en `:root`/`.dark`) — "premium-ish" pero sin identidad visual propia. El único rastro de violet era `--sidebar-primary: oklch(0.488 0.243 264.376)` en dark, saturado y aislado. Además, el usuario no podía elegir entre light y dark: la app renderizaba `:root` por default y no había setter de `.dark` en ningún lado. Después de F6: el usuario abre Settings → Apariencia, elige Claro/Automático/Oscuro con un selector visual de 3 cards, la preferencia persiste entre sesiones, y no hay flash de tema incorrecto al cargar. La paleta es oklch violet desaturado (chroma 0.005–0.12, hue 285°) — sutil, coherente, tipo Linear/Raycast.

## Qué se implementó

- **F1 — `@custom-variant` canonical + `color-scheme` property:** migrado `@custom-variant dark (&:is(.dark *))` → `(&:where(.dark, .dark *))` (canónico shadcn que matchea `<html class="dark">` y descendientes), agregado `color-scheme: light/dark` en `:root`/`.dark` para que scrollbars nativos, autocomplete y form controls respeten el modo en Tauri WebView2 y Capacitor Android. Archivos tocados: `src/index.css`.
- **F2 — Paleta violet desaturada + nuevos tokens semánticos:** `:root` y `.dark` reescritos con oklch hue 285° (chroma 0.005–0.025 para fondos/muted, 0.10–0.12 para `--primary`/`--ring`, light chroma menor que dark para compensar percepción). `--sidebar-primary` dark alineado con `--primary`. Tokens nuevos en ambos modos: `--background-deep` (backdrop modals), `--border-strong`, `--shadow-modal`, `--graph-project/area/resource/archive/default`. Mapeados en `@theme inline` para que Tailwind compile clases utility (`bg-background-deep`, `shadow-modal`, etc.). Archivos tocados: `src/index.css`.
- **F3 — `dark:` variants faltantes en badges semánticos:** agregadas variantes en `NoteCard.tsx:44` (semantic score badge violet), `SummaryL3.tsx:38` (border verde vertical), `ReviewBanner.tsx:34,46` (íconos Check verde + RotateCcw ámbar) — previamente invisibles o con bajo contraste en dark. Archivos tocados: `src/components/editor/NoteCard.tsx`, `SummaryL3.tsx`, `ReviewBanner.tsx`.
- **F4 — Token `--shadow-modal` en 5 modals:** reemplazado `shadow-[0_20px_40px_rgba(0,0,0,0.5)]` hardcoded → `shadow-modal`. La sombra varía por tema vía variable CSS (negro en dark, gris suave en light) — previamente generaba halo gris elefante anti-premium en light mode. Archivos tocados: `QuickCapture.tsx`, `CommandPalette.tsx`, `ObjectiveCreateModal.tsx`, `NoteLinkModal.tsx`, `ProjectCreateModal.tsx`.
- **F5 — `useTheme` hook + script anti-flash:** `src/lib/theme.ts` con types/constantes/helpers puros (`THEME_STORAGE_KEY = 'sm-theme'`, `DEFAULT_THEME = 'auto'`, helpers `readStoredTheme`/`systemPrefersDark`/`resolveTheme`/`applyTheme`). Hook con dos `useSyncExternalStore` (uno para `theme` desde localStorage, otro para `resolvedTheme` leyendo `documentElement.classList.contains('dark')` — DOM como fuente de verdad para evitar drift con script inline). Subscribe escucha `matchMedia` change + `storage` event + custom event `sm-theme-change`. Sin Provider/Context (mismo patrón que `useMediaQuery.ts`). Script IIFE síncrono en `<head>` antes de `main.tsx` con try-catch defensivo (Capacitor/Tauri pueden tirar en `localStorage.getItem`). Archivos tocados: `src/lib/theme.ts` (nuevo), `src/hooks/useTheme.ts` (nuevo), `index.html`.
- **F6 — Graph colors theme-aware:** `src/lib/theme-colors.ts` con maps `GRAPH_COLORS_LIGHT`/`GRAPH_COLORS_DARK` en hex strings por `ParaType` (Three.js no acepta oklch — descubierto en E2E porque Reagraph no lanzaba error, sólo caía a color default). `useGraph.ts` consume `useTheme().resolvedTheme` y dispara re-cómputo de nodos vía dep array. `KnowledgeGraph.tsx` construye un `ReagraphTheme` dinámico con `useMemo` overridando `canvas.background`, `node.label.color`, `edge.fill`, `arrow.fill` — corrige el rectángulo blanco hardcoded del `<GraphCanvas>` que se veía en dark mode. Archivos tocados: `src/lib/theme-colors.ts` (nuevo), `src/hooks/useGraph.ts`, `src/components/graph/KnowledgeGraph.tsx`.
- **F7 — ThemeSelector + Settings appearance section:** componente custom desde cero (no había patrón "card selector" previo, sólo `button.tsx` en `components/ui/`). Grid `grid-cols-1 sm:grid-cols-3` con 3 `<button>` `min-h-[112px]` (tap target 44 cumplido), estado activo con `border-primary bg-accent/40 ring-2 ring-primary/30`. Cada card: mini-preview CSS-only con placeholder bars + lucide icon (`Sun`/`Monitor`/`Moon`) + label. Preview "Automático" usa `clip-path: polygon(100% 0, 100% 100%, 0 100%)` para split diagonal light/dark en el mismo card. Sublabel reactivo en auto activo (`'oscuro'`/`'claro'` según `systemPrefersDark()`). `aria-pressed` para a11y. `settings/page.tsx` reescrito con `<section aria-labelledby>` y header "Ajustes". Archivos tocados: `src/components/settings/ThemeSelector.tsx` (nuevo), `src/app/settings/page.tsx`.

## Decisiones clave

| #   | Decisión                                                                       | Razón                                                                                                                  |
| --- | ------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| 1   | **Default = `auto` en vez de `dark`** (desvío SPEC D3)                         | App mostraba `:root` (light neutral) — `dark` forzado sería cambio impuesto sin opt-in. `auto` respeta sistema.        |
| 2   | **Scope expandido con fixes preexistentes** (F3-F4 + graph migration)          | Sin estos fixes light mode no es funcional. Mismo PR mantiene review coherente.                                        |
| 3   | **Graph colors en hex, no oklch** (desvío de "hardcodear oklch KEEP IN SYNC")  | Three.js no procesa oklch. Hex equivalentes con comentario KEEP IN SYNC en `theme-colors.ts`.                          |
| 4   | **Reagraph `theme` prop dinámico** (no estaba en plan original)                | Canvas blanco hardcoded del `<GraphCanvas>` fue regresión descubierta en E2E sólo en dark mode.                        |
| 5   | **`resolvedTheme` snapshot lee DOM** (no recalcula de localStorage+matchMedia) | Evita drift con el script inline que ya aplicó la clase antes de React montar.                                         |
| 6   | **Chroma 0.10–0.12 para primary, 0.005–0.025 para fondos**                     | Linear-style desaturado premium. 0.14+ se ve "vibrante". Light chroma menor para compensar percepción.                 |
| 7   | **Sin Provider/Context**                                                       | Tema es singleton DOM global. `useSyncExternalStore` es patrón correcto (mismo que `useMediaQuery`/`useOnlineStatus`). |

## Lecciones

- **Tailwind v4 `@custom-variant dark (&:is(.dark *))` sólo matchea descendientes, no el elemento mismo.** El selector canónico shadcn es `(&:where(.dark, .dark *))`. Usar el primero "funciona" sólo cuando el body hereda tokens vía `@apply bg-background` — bug latente que se manifiesta cuando algún componente lee tokens en `<html>` directo.
- **Tailwind v4 requiere `color-scheme: light/dark` en `:root`/`.dark` para que el browser nativo respete el modo.** Sin él, scrollbars en Tauri WebView2 / Capacitor Android, autocomplete dropdowns y form controls quedan con el color del OS, ignorando el tema de la app. No es opcional, no es cosmético.
- **Three.js (y por extensión Reagraph, react-three-fiber) NO procesa oklch strings — sólo hex, rgb, CSS color names.** El error es silencioso: el renderer cae a color default sin throw. Cualquier integración WebGL desde tokens oklch necesita una capa hex equivalente con KEEP IN SYNC documentado.
- **Reagraph `<GraphCanvas>` tiene fondo blanco hardcoded si no se le pasa el prop `theme`.** Construir un `ReagraphTheme` con `useMemo(dep: resolvedTheme)` extendiendo `lightTheme` base y overridando `canvas.background` + `node.label.color` + `edge.fill` + `arrow.fill`.
- **Cuando hay un script inline anti-flash, el `useSyncExternalStore` snapshot del `resolvedTheme` debe leer del DOM, no recalcular de `localStorage` + `matchMedia`.** El script ya aplicó la clase; recalcular en React introduce drift en edge cases (storage event mid-init, prefers-color-scheme cambia entre script y mount).
- **Los tokens fantasma se cuelan: 6 archivos referenciaban `--background-deep` y `--border-strong` pero los tokens no existían en `src/index.css`.** Tailwind v4 resuelve a `undefined` sin warning, los backdrops quedan transparentes. Auditar tokens nuevos con `grep -r "var(--token-name)"` antes de asumir que existen.
- **Sombras hardcoded `rgba(0,0,0,0.5)` en modals son trampa cross-theme.** Funcionan en dark (negro sobre oscuro = profundidad), generan halo gris elefante en light. Token `--shadow-modal` por tema desde el primer modal.
