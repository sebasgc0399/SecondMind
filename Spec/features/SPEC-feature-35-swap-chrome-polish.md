## Objetivo

El swap sidebar↔TopBar pasa de "funcionalmente correcto pero con artifacts visuales" a "fluido sin shifts ni pops". El usuario no percibe layout jumps en ningún escenario — el main mantiene su geometría estable mientras el chrome lateral entra y sale por overlay. Los toggles rápidos consecutivos (<200ms) no re-disparan animaciones espurias sobre componentes que ya estaban visibles. Cierra el alcance del modo hidden iniciado en F31.

## Qué se implementó

- **F35.1 — `useMountedTransition` justMounted + guard consumer:** Hook extendido con `justMounted: boolean` que persiste durante `durationMs` tras un mount real (mount inicial con visible=true Y re-mount tras exit completado). El consumer pasa `animateEntry={animateLayoutSwap && transition.justMounted}` en lugar de `!isExiting`, evitando aplicar `animate-in slide-in-from-X` a componentes que mantuvieron `shouldRender=true` durante toggles rápidos. 4 tests nuevos para justMounted + 6 tests existentes ajustados (10/10 pasan). Archivos tocados: `src/hooks/useMountedTransition.ts`, `src/hooks/useMountedTransition.test.ts`, `src/app/layout.tsx`.

- **F35.2 — sidebar overlay + padding-left transition main column:** Sidebar siempre overlay en desktop (`position:absolute inset-y-0 left-0 z-30`) para no empujar la inner column. Inner column con `transition-[padding-left] duration-200` derivado del estado target (256px cuando showSidebar, 0 cuando hidden), sincronizada al slide del sidebar. Outer flex con `overflow-x-hidden` defensivo. Sampling DOM 30ms validó: paddingLeft transitions smooth 0↔256px en ~250ms en ambas direcciones, mainW estable, ningún pop ni shift. Tablet 768 inline `w-16` (sin overlay) sin regresión. Mobile 375 (NavigationDrawer + MobileHeader + BottomNav) sin regresión. Archivos tocados: `src/app/layout.tsx`, `src/components/layout/Sidebar.tsx`.

## Decisiones clave

| ID  | Decisión                                                                                   | Razón                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| --- | ------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| D1  | `justMounted` con state-based + setTimeout durationMs (no useRef + 1-render-only)          | Si fuera 1-render-only via useRef + useEffect post-paint, un re-render del parent mid-animación leería justMounted=false y removería la clase animate-in en pleno vuelo, causando snap visual a translateX(0). Persistir el flag en state con timer matching la duración de la animación asegura que la clase se mantenga durante toda la animación incluso ante re-renders no relacionados.                                                                                                                                                                                                                     |
| D2  | Sidebar always-floating en desktop (Option B), no conditional floating durante swap window | El conditional `floating={isExiting \|\| justMounted}` del plan original causaba timing issues con la transition de padding-left: cuando `floating` flippeaba a false al final del swap, el sidebar volvía a flex inline y la transition de padding-left arrancaba desde el estado erróneo. Option B (`floating={!isMobile && !isTablet}`) elimina la lógica condicional y mantiene el sidebar overlay permanentemente en desktop. Trade-off: mayor distancia del modo "flex declarativo" original; beneficio: menos code paths, menos riesgos de edge cases con timing. Tablet sigue inline w-16 (sin overlay). |
| D3  | TopBar overlay vertical fuera de scope F35                                                 | Sampling DOM confirmó que el TopBar genera un pop de 48px al unmount, pero la magnitud es subliminal vs los 256px del sidebar. El slide-out-to-top de F33 lo enmascara visualmente. Diferido a backlog si post-deploy se observa como visible y molesto.                                                                                                                                                                                                                                                                                                                                                         |

## Lecciones

- **Animation flags deben persistir durante toda la duración de la animación, no derivarse de comparaciones instantáneas.** Si una clase `animate-in/animate-out` se condiciona a un flag de "estoy montando/saliendo ahora", el flag debe sobrevivir re-renders del parent mid-animación. State + setTimeout matching la duración funciona; useRef + 1-render-only no funciona porque el segundo render ya no tiene el flag y la clase se cae mid-vuelo. Complementa la lección F32.3 sobre `useLayoutEffect` (la setState debe correr ANTES del paint inicial); aquí la lección es sobre la VENTANA de vida del flag tras ese paint inicial. Aplicable a cualquier hook futuro de transición animada (modal entry, drawer slide, tab swap) que use clases CSS condicionales en lugar de Web Animations API.

- **Layout overlay siempre-on simplifica edge cases vs conditional-on.** Cuando un componente cambia de `position:static` a `position:absolute` y vuelve, las transitions sobre propiedades del flex container (padding, gap, width) sufren de "estado inicial inconsistente" en el momento del flip. Mantener el overlay activo permanentemente cuando aplica (ej. desktop), aunque distancie del modo flex declarativo, elimina la clase de bugs donde la transition arranca desde la geometría equivocada. La trade-off (menos "puro") es preferible a la fragilidad de timing.

- **`overflow-x-hidden` defensivo en outer container con overlays animated-out.** Cualquier elemento con `position:absolute left:0` que anime `translateX(-100%)` durante 200ms queda parcialmente fuera del viewport con su `left:0` declarado. Sin `overflow-x-hidden` en el contenedor padre, navegadores pueden renderizar scrollbar horizontal momentáneo. Belt-and-suspenders sin costo.

- **Sampling DOM 30ms con dispatch in-page de eventos teclado, no via tool round-trips.** Tool calls de Playwright tienen ~3s de round-trip cada uno, así que un "fast toggle <200ms" via 2 calls separadas de `browser_press_key` termina siendo de ~9s — no valida lo que se quiere. Para validar timing fino, dispatch de KeyboardEvent dentro de `browser_evaluate` con setTimeout interno (in-page) garantiza el intervalo deseado.
