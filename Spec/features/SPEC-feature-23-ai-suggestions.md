## Objetivo

Manifestar la AI invisible que ya corre en background (`autoTagNote`, `generateEmbedding`) vía un banner inline en el editor que sugiere acciones contextuales sobre la nota con accept/dismiss explícito (P6: AI copiloto, no piloto). F23 absorbió F25 del roadmap — sin la sugerencia AI persistida, F23 sería infra UI vacía con solo la heurística client-side.

Dos triggers en este SPEC:

- **A. Sugerencia AI persistida (server-side):** CF `autoTagNote` extendida retorna `suggestedNoteType` cuando la heurística semántica detecta encaje en `literature`/`permanent` con confianza ≥0.7.
- **B. Sugerencia heurística (client-side):** sin CF, computada en cada render — "promover a `permanent`" cuando `outgoingLinkIds.length > 3` Y `summaryL3` truthy Y `noteType !== 'permanent'`.

## Decisiones clave

| Decisión                                                                                                   | Razón                                                                                                                                                                                                                                                                             |
| ---------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `dismissedSuggestions` + `suggestedNoteType` + `noteTypeConfidence` SOLO en Firestore (no schema TinyBase) | Persister F12 emite `setDoc(merge:true)` con la row entera tras `setPartialRow`; durante la ventana entre primera hidratación y primer onSnapshot real, sentinel default sobrescribiría valores que la CF escribe directo. Excluir del schema → persister no toca el campo nunca. |
| `arrayUnion` vía `updateDoc` directo (NO `repo.update`)                                                    | `setDoc(merge:true)` del factory NO interpreta el sentinel `arrayUnion` correctamente. Bypass intencional al patrón factory para garantizar atomicidad cross-device.                                                                                                              |
| `onSnapshot` directo en `useEffect` del hook (no cache module-level)                                       | Solo 1 editor abierto a la vez = 1 listener; cleanup automático con remount `key={noteId}`. Promote a cache module-level si emerge un segundo consumer.                                                                                                                           |
| Optimistic local `Set<string>` en el hook (no en el componente)                                            | Una sola fuente de verdad evita desync entre hook y componente. Reset on remount aceptable porque para entonces el `arrayUnion` ya confirmó por onSnapshot (idempotente).                                                                                                         |
| `role="region"` sin `aria-live` en el banner                                                               | Información persistente, no notificación efímera. `aria-live="polite"` provocaría doble anuncio screen reader si la sugerencia reaparece tras race onSnapshot.                                                                                                                    |
| Prompt caching `cache_control: { type: 'ephemeral' }` en CF Anthropic                                      | System prompt extendido a 150 palabras × N notas/día sin caching es costoso. Con caching el segundo+ call dentro de 5min TTL es 10% del costo de input.                                                                                                                           |
| `noteTypeConfidence` clamp manual post-tool-use                                                            | Anthropic enforce `enum` pero `minimum`/`maximum` son hints, no estrictos. `Math.min(1, Math.max(0, ...))` antes de escribir al doc.                                                                                                                                              |
| Threshold confianza ≥0.7 para activar sugerencia AI                                                        | Filtra ruido visual con sugerencias dudosas. Conservador, ajustable si emerge feedback.                                                                                                                                                                                           |

## Lecciones

- **`arrayUnion` requires `updateDoc`, not `setDoc(merge:true)`.** Cuando un campo del doc se muta atómicamente por múltiples writers (cross-device, paralelos rápidos), debe ir vía `updateDoc(ref, { campo: arrayUnion(value) })` directo. Documentar como excepción al patrón factory cuando se introduzca un nuevo helper similar.
- **Persister F12 + campos escritos directo desde CF requiere coordinación.** Si la CF escribe un campo a Firestore Y ese campo está en schema TinyBase con default sentinel, el persister al hidratar tras `setRow` puede sobrescribir el valor de la CF con el sentinel durante una ventana de carrera. Mitigación canónica: NO incluir esos campos en schema TinyBase; el hook UI lee via `onSnapshot` directo en `useEffect` con cleanup automático en remount.
- **`role="region"` para banners persistentes; `aria-live="polite"` solo para efímeros.** Si una sugerencia aparece, dismissea, reaparece por race onSnapshot, `aria-live` provoca doble anuncio screen reader. Información persistente → `role="region"` + `aria-label`. Notificación efímera (DistillLevelBanner) → `role="status" aria-live="polite"`.
- **Prompt caching ephemeral en CFs Anthropic con system prompt no trivial.** `cache_control: { type: 'ephemeral' }` reduce 90% costo input tokens en invocaciones subsecuentes (TTL 5min). Aplicable a cualquier futura CF con system prompt >100 palabras.
- **Anthropic enforce `enum` en JSON Schema, NO `minimum`/`maximum`.** Validar manualmente en CF tras tool_use con clamp explícito (`Math.min(max, Math.max(min, value))`). Aplicable a cualquier campo numérico con rango.
- **Hook como única fuente de verdad del optimistic state evita desync con componente.** Si dos componentes (o hook+componente) mantienen Sets paralelos del mismo dismiss, divergen en remounts y reconciliación. El hook expone handlers ya integrados con el repo + state propio; el componente solo invoca y renderea.
- **Heurística client-side simple (≥4 wikilinks + summary L3) puede convivir con AI server-side con dedup explícito.** Para sugerencias que pueden derivar de ambas fuentes (ej. "promote to permanent"), priorizar AI (más contexto) y filtrar la heurística si la AI ya cubrió. Patrón aplicable a cualquier futuro signal mixto.
