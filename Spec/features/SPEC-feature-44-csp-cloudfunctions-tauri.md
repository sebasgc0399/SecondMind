## Objetivo

Búsqueda híbrida en `/notes` de la app Tauri quedaba inservible: cualquier query devolvía "Sin resultados" porque el fetch a `embedQuery` (Cloud Function callable) era bloqueado por la CSP del WebView2 antes de salir. F44 desbloquea la invocación de callables Firebase agregando el hostname al `connect-src`.

Root cause: la CSP de `tauri.conf.json` enumeraba los servicios Firebase (`identitytoolkit`, `securetoken`, `*.googleapis.com`, `*.firebaseio.com`, `*.firestore.googleapis.com`) pero omitía `cloudfunctions.net`, donde viven las CFs callables invocadas vía `httpsCallable`. Bug Tauri-only: PWA web (sin CSP custom) y Capacitor Android (sin policy) no aplican esta restricción. Cero invocaciones en Cloud Logging confirmó que el request nunca salía del cliente — el lado servidor estaba sano. F44 es hermana de F39 (`worker-src` para Reagraph) — mismo perfil: CSP de Tauri omite una capa que la PWA recibe por default. Bug latente desde F3 (búsqueda híbrida) porque `embedQuery` fue la primera y única callable hasta esta feature.

## Qué se implementó

- **F44.1 — CSP `connect-src` con `cloudfunctions.net`:** se agregó `https://us-central1-secondmindv1.cloudfunctions.net` al `connect-src` de Tauri en el bloque Firebase (post `securetoken`, pre `www.googleapis.com`) para desbloquear `httpsCallable(embedQuery)` desde la app desktop. Validado en `tauri:dev` con query "inteligencia artificial" → 3 resultados temáticos con cosine similarity 37%/32%/31% y header "✨ Notas temáticamente similares". Archivos tocados: `src-tauri/tauri.conf.json`.
- **F44.2 — Update gotcha CSP Firebase:** se editó in-place el gotcha "CSP Firebase explícito en `tauri.conf.json`" agregando `cloudfunctions.net` a la enumeración + nota sobre que aplica a callables (`httpsCallable`) y NO a triggers Firestore/scheduled (que corren server-side). Extensión de enumeración canónica, no gotcha nuevo. Anchor preservado → ESTADO-ACTUAL.md sin update del índice. Archivos tocados: `Spec/gotchas/tauri-desktop.md`.
- **F44.3 — Release coordinado v0.2.10:** skill `release-ecosystem` bumpeó los 5 archivos atómicamente (`package.json`, `package-lock.json`, `src-tauri/Cargo.toml`, `Cargo.lock`, `tauri.conf.json`), hosting deployado pre-tag a https://secondmind.web.app, tag `v0.2.10` disparó CI `release.yml` (9.08 min, ambos jobs success): GitHub Release con MSI + NSIS + `latest.json` firmados, APK desplegado a Firebase App Distribution grupo `owner`.

## Lecciones

- **Callables Firebase NO se cubren por `*.googleapis.com`** — viven en `cloudfunctions.net` (TLD separado). Cualquier nueva callable agregada al cliente exige update explícito del `connect-src` de Tauri. Triggers (`onDocumentCreated`, `onSchedule`, etc.) NO lo requieren porque corren server-side y no salen del WebView. Distinguir el subtipo de CF al agregarla.
- **Bugs Tauri-only de CSP quedan latentes hasta que un flow los invoca** — F44 fue el segundo caso del repo (F39 fue el primero con `worker-src` para Reagraph). PWA web y Capacitor Android no aplican CSP custom, así que el bug solo se manifiesta en desktop. Patrón canónico: cualquier nueva integración externa (API third-party, CF callable, worker blob, CDN font, OAuth endpoint) debe testearse explícito en `tauri:dev` antes del próximo release coordinado.
- **`tauri-plugin-single-instance` puede sabotear smoke tests desktop** — si la app instalada (MSI/NSIS) está corriendo en el tray cuando se lanza `npm run tauri:dev`, el plugin detecta segunda instancia y abre la ventana de la app ya instalada en lugar de la dev. DevTools muestra entonces el bundle viejo + CSP viejo. Síntoma confuso: "el fix no funciona" pero el error muestra el CSP de la versión instalada (bundle JS con hash producción `index-XXXXX.js` en vez de scripts dev de Vite `http://localhost:5173/src/main.tsx`). Mitigación obligatoria pre-smoke-test desktop: cerrar completamente desde el tray (right-click → Salir/Quit, no solo cerrar ventana — close-to-tray mantiene proceso vivo).
- **Cero invocaciones en Cloud Logging es señal positiva para CSP-bloqueado** — al diagnosticar fallos de Cloud Functions desde el cliente, antes de hipotetizar problemas del lado servidor, verificar logs vía `mcp__firebase__functions_get_logs` (o `firebase functions:log`). Cero invocaciones recientes confirma que el request nunca salió del cliente → bug en cliente (CSP, network, auth). Errores 500/timeouts confirman fallo servidor. La discriminación toma segundos y evita debugging en el lado equivocado.
- **Font OTS error en Tauri (`invalid sfntVersion: 1008813135` = `0x3C21444F` = `<!DO`)** — el WebView2 está descargando HTML (probablemente fallback SPA) en lugar del woff2. Detectado en consola durante diagnóstico de F44 pero NO relacionado al CSP fix. Root cause confirmado en build v0.2.10: warnings de Vite indican que `geist-latin-wght-normal.woff2`, `geist-cyrillic-wght-normal.woff2` y `geist-latin-ext-wght-normal.woff2` "didn't resolve at build time, will remain unchanged to be resolved at runtime" — el bundler dejó las referencias literales sin copiar los archivos al output. Caen al `font-sans` fallback (sin impacto visual aparente). Candidato a feature de polish independiente.
